{Forecast Visualization}

Hello and welcome to my forecast visualization.

The goal of this project was to visualize forecasted data so that the engineer or user can analyse and understand the machine learning (ML) model's capabilities and limits. Besides, I also wanted to learn more about D3.js.

The data

The data you see in this project is the bikesharing dataset from Kaggle. The ML goal was to build a forecasting model that predicts the demand of bikes for a given date and hourly time. Below you can see the target, prediction as well as error as line plots. Furthermore, additonal features - besides datetime features - that the model used are displayed as barplot - Monday to Friday and weekends - and horizontal line plots - workingday/no-workingday and holiday/no-holiday.

The machine learning model

The ML model is a basic Random Forest (RF) model and was implemented using the ML library sklearn and its great pipeline structure. The best RF model was selected via grid search. To ensure that the grid search runs with the correct forward moving splits, because we have a time series data at hand, the ForwardTimeSeriesSplit was used. After the best model was selected, the question was how good the final model is. For this, a forward-test was run using the ForwardTimeSeriesSplit again and a holdout dataset, which the grid search hasn't seen before. The resulting predictions from the forward test are visualized below. The model itself isn't perfect, and can and should be further optimized. The visualizations below can be used to analyse the forecasts and formulate new questions and hypotheses to further improve the model.
